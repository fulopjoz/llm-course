Validation checklist

Check AI-assisted analysis before using it in a project decision.

The validation rule

1. Data: does the answer match the data?
2. Logic: does the reasoning make sense?
3. Scope: is the conclusion too broad?
4. Decision: is the output useful for the actual project question?

Data validation

  • Are the numbers mentioned by AI actually present?
  • Did AI invent values, products, categories, or averages?
  • Are missing values clearly mentioned?
  • Are units clear?
  • Are categories consistent enough to compare?

Reasoning validation

  • Is AI comparing similar products?
  • Did AI confuse one example with a trend?
  • Did AI confuse correlation with causation?
  • Did AI make health claims the dataset cannot support?
  • Did AI explain uncertainty?

Red flags

Be careful when AI says: clearly proves, definitely, the healthiest, the worst, always, never, or gives exact averages that you did not calculate.

Safer language

  • In this sample...
  • Based on available fields...
  • A possible pattern is...
  • This requires verification because...
  • The dataset does not prove...

Student task

Take one AI-generated answer and mark each important sentence as supported, plausible but not proven, unsupported, too broad, or needing rewrite.
