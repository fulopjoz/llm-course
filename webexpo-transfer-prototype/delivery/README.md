# Delivery strategy

This folder explains how to deliver the mini-course **AI for student projects: from messy information to reliable decisions** to students.

## Recommendation

Use a simple delivery model:

1. **Primary delivery:** Microsoft Teams or Google Drive.
2. **Source of truth:** GitHub repository.
3. **Optional later delivery:** static website.
4. **Optional future demo:** guided web app.

The first version should not require a custom web app. Students should be able to open the materials, follow instructions, inspect the CSV, use one AI chat assistant, validate the output, and submit a short decision brief.

## Why Teams or Google Drive first

Teams and Google Drive are practical because:

- students already know how to open files from course platforms;
- no GitHub account is required;
- no programming environment is required;
- instructions can be shared as PDFs;
- the CSV can be opened in Excel, Google Sheets, or LibreOffice;
- the lecturer can collect submissions through normal course workflows.

## Role of GitHub

GitHub is the source of truth for maintaining the course materials.

Use GitHub for:

- version control;
- lecturer review;
- optional technical students;
- keeping Markdown, CSV, and Python files organized;
- later publishing as a static website.

Do not require students to use GitHub for the main activity.

## Student requirements

Required:

- spreadsheet tool: Excel, Google Sheets, or LibreOffice Calc;
- one AI chat tool: ChatGPT, Gemini, Claude, or similar;
- provided CSV file;
- decision brief template or instructions.

Not required:

- GitHub;
- Python;
- Codex;
- Claude Code;
- GitHub Copilot;
- Cursor;
- paid AI tools.

## Optional static website later

A static website can make the course easier to browse. This can be created later with GitHub Pages, MkDocs Material, or Google Sites.

The static site should only mirror the same content. It should not replace the simple downloadable student pack.

## Optional web app later

A web app is not needed for the first version. It may become useful later as a guided demo, for example a decision-brief builder that walks students through prompts and validation.

Possible tools for later prototypes include Claude Artifacts or Lovable. These should remain optional demos, not the main course delivery method.

## Best first delivery package

Prepare a student package with:

- start-here guide;
- dataset CSV;
- student instructions;
- one-page checklist;
- prompt pack;
- example decision brief;
- optional technical folder;
- teacher notes kept separate.

The student package can be shared as a Drive folder, Teams folder, or ZIP file.
