# 08 - Teacher notes

## Purpose

This activity is designed as a transfer output from WebExpo 2026 back to the UCT Prague Technology Skills for Project Managers course.

It gives students a practical way to use AI tools with real-world data while emphasizing validation, uncertainty, and project communication.

## Why it fits the course

The Technology Skills for Project Managers course covers a broad set of practical digital skills, including AI, data, SQL, low-code/no-code tools, cloud, and peer learning.

This activity fits because it combines:

- AI literacy;
- data inspection;
- spreadsheet work;
- optional coding;
- project decision-making;
- communication for mixed teams;
- responsible use of AI.

## Why it works for mixed-specialization students

The main route requires no coding. Students only need a spreadsheet and one AI chat tool.

The dataset is food-product data, which is understandable across disciplines:

- chemistry students can think about salt, sugar, fat, ingredients;
- business students can think about products, categories, consumer decisions;
- IT students can think about data quality and automation;
- design students can think about communication and presentation;
- bioinformatics or data students can think about structured analysis and validation;
- project-management students can think about decisions, assumptions, and risks.

## Connection to WebExpo topics

The activity connects to WebExpo themes such as:

- AI and messy data;
- reducing analytics noise;
- preparing projects or codebases for AI agents;
- AI in product/project management;
- evidence-led decision-making;
- human validation of AI output.

The transfer is not a conference summary. It is a reusable classroom exercise inspired by those topics.

## Minimum tools

Required:

- Excel, Google Sheets, or LibreOffice Calc;
- ChatGPT, Gemini, Claude, or a comparable AI chat assistant;
- provided CSV file.

Not required:

- GitHub;
- Python;
- Claude Code;
- Codex;
- GitHub Copilot;
- Cursor;
- paid AI tools.

Optional extension:

- Python + pandas;
- Codex, Gemini Code Assist, GitHub Copilot, Cursor, or Claude Code for code explanation and review.

## Suggested class format

This can be run as a 35-40 minute mini-lesson.

1. Introduce the workflow.
2. Open the dataset in a spreadsheet.
3. Ask AI for risks and possible patterns.
4. Validate one AI claim manually.
5. Rewrite the finding cautiously.
6. Draft a short decision brief.
7. Discuss optional technical extension.

## Assessment criteria

Students should be assessed on the workflow, not on producing the most impressive AI answer.

Good student work includes:

- a clear project question;
- evidence of manual data inspection;
- a useful AI prompt;
- validation of at least one AI claim;
- at least three limitations or risks;
- cautious wording;
- a short decision brief;
- an AI-use log.

## What to avoid

Students should not be rewarded for:

- copying raw AI output;
- broad claims without evidence;
- treating the dataset as medical truth;
- using expensive tools as a substitute for understanding;
- focusing only on coding when the class is mixed-specialization.

## Suggested deliverables

Each student or small group submits:

1. One-page decision brief.
2. AI-use log.
3. Validation checklist.

Optional technical students may also submit:

- generated category summary CSV;
- short note on how they used a coding assistant;
- one limitation of the script.

## Lecturer review note

The activity is intentionally beginner-first. The optional coding track exists only to connect the lesson to AI-agent/codebase themes from WebExpo and to provide a deeper path for technical students.
